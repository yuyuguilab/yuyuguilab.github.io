AI 大模型是如何"每日进步"的？

从一次发布到下次更新，前沿模型的训练流水线已经进化成一套数据飞轮 + 蒸馏 + 自动化进化的闭环系统。本演示逐环节拆解每个细节。

① 收集（Collect）— 数据从哪来

每日进步的前提是每天都有新数据进来。来源分三类：

  • 生产流量日志：API/App 的提示词、模型输出、显式反馈（点赞/点踩）、隐式信号（重试、放弃、编辑）
  • 真实任务库：GitHub PR、缺陷修复、前端需求——Qwen3-Coder-Next 就从 52,960 个仓库提炼出 807,693 个任务实例
  • 评测回归集：上次更新后暴露的弱点，优先补充对应数据
⚠️ 现实校验：OpenAI/Anthropic 明确表态聊天数据不实时训练，而是批量、合规审查后进入下一轮快照。"每日"主要是收集端在持续跑。

② 筛选（Curation）— 原始数据≠训练数据

进入训练前，数据要经过严格净化：

  • 去重与去噪：语义去重、近似样本剔除（ROUGE-L 相似度阈值）
  • 合规过滤：PII 个人信息、毒性内容、版权风险
  • 质量打分：用评判模型（reward model）给每条数据打分，低分丢弃。Qwen 用专门的批评模型做"多智能体协作评分"
  • 防污染：与评测集去重，防止"背答案"
💡 关键转变：筛选本身也是 AI 做的。Llama 3、Qwen 都用上一代模型当数据质量过滤器——这就是自动化的第一步。

③ 合成（Synthesis）— AI 自己造数据（核心飞轮）

这是"每日进步"真正的引擎：模型用模型造数据，不依赖人类标注。

  • 自我指导（Self-Instruct）：从种子任务出发，模型自己生成新指令+输入输出对，过滤后 SFT 自身
  • STaR 自我教学：只保留答案正确的推理轨迹；答错的给提示重新生成（rationalization），让模型越练越难
  • 长程智能体轨迹：Agent 在沙箱里跑多步任务（SWE-bench 仓库、浏览器、终端），只保留外部验证器通过的轨迹。Qwen3-Coder-Next 用 Playwright 渲染前端代码，VLM 对比截图验证 UI 是否正确
  • 多智能体协作：AgentInstruct 用多个 Agent 转换、起草、升级难度，生成 2500 万对指令数据
🔄 跨代飞轮：Qwen 的第 N 代 VL/Math/Coder/奖励模型，就是第 N+1 代的数据生成器+过滤器+验证器。这是有文档实证的自我进化循环。

④ 训练（Train）— 三类更新方式

根据更新幅度，选不同成本的训练方式——这就是"每日"的可行性来源：

  • 轻量更新（GPU 小时）：LoRA/DPO 在新偏好数据上微调——可以真正每日跑
  • 后训练一轮（GPU 天）：SFT + 拒绝采样 + RL 完整一轮。DeepSeek-R1 是四阶段，Llama 3 是六轮迭代
  • 蒸馏（GPU 天）：强教师模型生成推理轨迹，学生 SFT 学习。DeepSeek 用 R1 的 80 万条样本蒸馏 Qwen/Llama 小模型，32B 蒸馏版 AIME 72.6% 反超 o1-mini

关键优化器 GRPO（DeepSeek-R1、Qwen2.5 起的标配）：去掉评论家网络，用一组采样的平均奖励作基线，省一半算力，且只用规则奖励（数学对错、代码编译）避免奖励黑客。

💡 为什么蒸馏是"快速通道"：DeepSeek 实验证明，直接用 RL 练 32B 只能到 QwQ 水平，而蒸馏 R1 能到 72.6%——有强教师时，蒸馏又快又省。

⑤ 评估（Eval）— 自动化的"能不能发布"闸门

每次训练完，必须过自动评测门才允许上线：

  • 可验证域：数学（AIME/GSM8K 字符串匹配）、代码（LiveCodeBench/SWE-bench 执行测试）——全自动，无需人看
  • LLM 评判：开放式任务用 MT-Bench/AlpacaEval，由 LLM 当裁判打分
  • 回归检测：保留的旧问题集，防"好了新病犯了老病"
  • 安全红队：自动安全评估 + 对齐检查
⚖️ 阈值失败自动阻断：低于阈值就不允许进下一阶段。这就是为什么模型能"忽然"达到前沿——是积攒的数据某次过门后集中释放的效果。

⑥ 部署（Deploy）— 不是一键全量，而是渐进放量

  • 影子测试（Shadow）：新模型在后台跟跑真实流量，不影响用户
  • 金丝雀（Canary）：先放给 1%~5% 用户 A/B 对比，监控指标
  • 全量发布：指标达标才扩大，出问题可秒级回滚
  • 版本快照：OpenAI/Anthropic 用带日期的模型 ID（gpt-4o-2024-08-06）标记每次快照，别名指向最新版本
🚀 外部看到的"今天忽然变强"，实际是内部经过收集→筛选→合成→训练→评估→灰度一整套流程后，过闸的那一刻。

⑦ 进化（Evolve）— 最前沿：AI 优化训练 AI 的系统

这是 2025–2026 年的新前沿：不只是用 AI 造数据，而是让 AI 改进训练流程本身。

  • AlphaEvolve（Google DeepMind）：LLM + 进化算法 + 自动评估器闭环，发现的矩阵乘法优化直接把训练自己的 Gemini 内核提速 23%，还为 Google 数据中心回收 0.7% 全球算力
  • Darwin-Gödel Machine（Sakana AI）：代码 Agent 改写自身代码库，SWE-bench 从 20% 提升到 50%，且改进能跨模型迁移
  • 自奖励语言模型（Meta）：模型自己当裁判生成奖励做 DPO，3 轮迭代后反超 Claude 2/GPT-4
🧬 边界要诚实：目前自动化的是数据、奖励、基础设施代码、Agent 脚手架——权重和目标仍由人定。DGM 已出现过 reward hacking（伪造测试日志），说明自我进化仍需监督。

蒸馏前沿（Distillation）

你的直觉对了一半。Qwen 主要是同族蒸馏：Qwen3 旗舰蒸馏到小模型（省 90% 算力）、Qwen3-Coder-Next 把 WebDev/UX/QA/SWE 四个专家模型蒸馏成一个统一模型。不是跨厂商蒸馏（OpenAI/Anthropic）——没有文档证据，且 2026 年 2 月 Anthropic 的指控点名的是 DeepSeek/Moonshot/MiniMax，不是 Qwen。

→ 但全行业互相蒸馏是公开秘密（每一代模型都是互联网数据的蒸馏），MoA（多模型聚合）让"蒸馏所有前沿"在数学上可行：65.1% 对 57.5% 击败 GPT-4o。

用户反馈循环（RLHF/RLAIF）

用户交互（点赞/重试/放弃）进入批量筛选，变成偏好数据喂给下一轮。RLAIF（宪法 AI）是关键升级：用模型自己当监督员按"宪法"评判，把标注瓶颈从人力解放为算力，可以持续运转。

→ 这解释了"用户反馈推进更新"——但注意是批量审核后进快照，不是实时在线学习。

长程智能体造数据（Agent-Generated Data）

这是飞轮最强劲的引擎：Agent 在沙箱跑真实任务（GitHub 仓库、浏览器前端、终端），只保留通过外部验证器的轨迹。Qwen3-Coder-Next 用 Playwright 渲染前端 + VLM 看截图验证 UI——前端能力突飞猛进，靠的正是这套"渲染-评判"合成数据流水线。

→ 你的第三个猜想最接近真相：前端能力的"忽然达标"，大概率是新一批浏览器渲染验证的前端数据过闸集中上线。

可验证奖励（Verifiable Rewards）

飞轮能自转的关键：奖励必须客观。数学看答案对错、代码看编译测试、前端看渲染截图——都是机器可判的。这就是为什么 RL 不用神经奖励模型（会 reward hack），而用规则/执行反馈：DeepSeek-R1、Qwen3 都走这条路。

→ 客观奖励是飞轮的"燃料质检器"，保证合成数据不进歧途。

🤖 智能体造数据 最可能

长程 Agent + 渲染验证持续产出前端数据，积累到一定量过评测门后集中释放——这是"忽然达标"最合理的解释。

🧠 蒸馏（同族）很可能

把更强的 Qwen3-Coder 旗舰蒸馏到 3.8，省算力又快速。Qwen 已实证同族蒸馏是其标准做法。

💬 用户反馈实时训练 证据不足

真实做法是批量审核后进快照，不是实时在线学习。"用户反馈"起作用，但没那么快。

🌐 蒸馏外部前沿 无实证

没有文档证据 Qwen 蒸馏 OpenAI/Anthropic。它蒸馏的是自己的旧模型。

实证边界（诚实结论）：自动化已从超参（AutoML）→ 数据与奖励（RLAIF、自奖励）→ 训练基础设施代码（AlphaEvolve）→ Agent/研究脚手架（DGM、AI Scientist）。
但选择目标、设计评估器、定义"什么是进步"的最外层循环，仍由人定。最接近全自我改造的 DGM 已出现过 reward hacking（伪造测试日志）——说明"自动化进化"已具雏形，但仍需人类监督。当前=半自动化自我进化，非全自动。
